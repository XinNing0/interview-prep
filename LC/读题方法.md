# 做题清单 · 读题防错 + 选方法

> 每道题**开写前**扫一眼这张清单。前面几步不写代码，用嘴/脑子过。
> 花 1-3 分钟走流程，省后面 10 分钟返工。

---

## Part 0 · 读题防错技巧（针对读漏 / 读颠倒）

读题时**物理辅助**，不要用眼睛干扫：

1. **光标/手指指着逐字读** —— 眼睛跟着一个锚点走，减少跳字、颠倒。
2. **小声念出来** —— 耳朵帮眼睛校对，念到不对会察觉。
3. **一次只读一句** —— 读一句→停→理解→再下一句。别整段扫。
4. **先看 Example，再回头看文字描述** —— 例子最直观，先建框架再读描述，不易读偏。
5. **关键约束当场抄下来**（见下）——别指望脑子记住 k / "最多" / "连续"。
6. **允许自己读慢** —— 不跟任何人比读题速度。慢而准 >> 快而返工。

## Part 1 · 读题四步（读到就抄进注释）

```python
# 输入: ______（数组？字符串？数字？0/1数组？有序吗？）
# 返回: ______（长度？个数？下标？布尔？最大/最小值？）
# 约束: ______（每个数字、每个"最多/最少/不超过X"都圈出来！）
# 动作: ______（把每个操作用一句话拆开）
```

**"约束"这行最重要** —— 今天漏看 424 的 k 就是栽在这。
读到 **数字 / "最多" / "最少" / "不超过" / "连续" / "有序" / "O(n)要求"** 全部抄下来。

## Part 2 · 复述 + 找题眼

- **复述**：用**自己的话**讲一遍这题在干嘛。讲不清 = 没读懂，回去重读。
  （这也是面试第一步 "restate the problem"，一举两得）
- **找题眼**：问自己 —— **"哪个条件抽掉，题目就不成立？"**
  那个条件通常就是解题钥匙。（我在 424 上自己悟出来的招）

## Part 3 · 选方法（两个判断 + 信号表）

**判断 A：要不要存东西？**
- 问"在不在 / 去重" → **set**（自带去重！）
- 数"几个 / 出现几次 / 多种各自的次数" → **dict**
- 只关心"一种东西的数量"（如0的个数、和）→ **一个变量**就够，别上 set/dict

**判断 B：遍历用下标还是取值？**
- 要算长度 `j-left+1` / 比前后 / 动指针 / 返回下标 → **下标** `range(len)`
- 只要值 → `for x in nums`
- 都要 → `for i, x in enumerate(nums)`

**信号表（输入特征 → 方法）：**
| 特征 | 方法 |
|---|---|
| 有序数组 + 查找 | 二分 / 双指针对撞 |
| 连续子数组/子串 + 最长/最短 | 滑动窗口 |
| 子数组 + 和=某值/整除/个数（可能负数）| 前缀和 + 哈希 |
| 子数组 + 最大和/最大积（有负数）| DP (Kadane) |
| 数值连续（非位置）+ 无序 | set（只从起点数）|
| 字符串 + 计数字符 | dict 计数 / set 判断 |
| 链表 | 双指针（快慢）/ 反转 |
| 树 | DFS 递归 / BFS 队列 |
| 网格连通块 | BFS / DFS |
| 所有组合/排列/子集 | 回溯 |
| 最优解 + 重叠子问题 | DP |
| 下一个更大/更小 | 单调栈 |
| Top K / 第K大 | 堆 |

## Part 4 · 先暴力，再优化

1. 先想**暴力解**（通常双重循环 O(n²)），先有个能跑的。
2. 问：暴力慢在哪？有什么重复计算？
3. 针对浪费优化：
   - 重复查找 → 哈希表记历史（O(1)查）
   - 有序没利用 → 二分 / 双指针
   - 重复算子数组和 → 前缀和
   - 窗口反复重算 → 滑动窗口（指针不回退）

## Part 5 · 完整流程（贴墙上）

```
1. 读题（光标指 + 出声念 + 一次一句）
2. 先看 Example 对照，确认没读偏
3. 四步法：输入？返回？约束？动作？（约束抄进注释）
4. 复述：用自己的话讲一遍（讲不出=回去重读）
5. 找题眼：哪个条件抽掉题就垮？
6. 选方法（判断A set/dict + 判断B 下标/取值 + 信号表）
7. 先说暴力，再想优化
8. 才开始写
```

## 我的常错清单（写前扫一眼，别再犯）

- [ ] 用了 `i`（下标）却当成值 `nums[i]` 用？
- [ ] 找最大/最长，忘了用 `result = max(result, ...)` 记录？只返回了最后值？
- [ ] 循环变量和数组重名（`for nums in nums`）？→ 用单数 `num`
- [ ] dict 用了 list 的 `.append()`？→ dict 是 `d[k]=v`
- [ ] 漏看题目给的约束（k、最多、有序…）？
- [ ] for 循环里多写了 `i += 1`？（for 自动推进，那是 while 的习惯）
- [ ] Python 2 的整除坑？→ 用 **Python3**，`/` 才有小数
- [ ] set 版遍历了原数组不是 set？→ 遍历 `set`，否则重复元素拖垮 O(n)
- [ ] `count[k]` 取不存在的 key 报错？→ 用 `.get(k, 0)` 或 `defaultdict(int)`

## Java → Python 省力工具

| 要做的事 | Python 一行 |
|---|---|
| 计数 | `Counter(nums)` 或 `d.get(k,0)+1` |
| 取前 K 大/小 | `heapq.nlargest(k, ...)` / `nsmallest` |
| 按属性排序 | `sorted(x, key=lambda i: i[1])` |
| 求交集/并集/差集 | `a & b` / `a \| b` / `a - b` |
| 拼字符串（无空格）| `"".join(list)` |
| 拆数字每一位 | `for ch in str(n): int(ch)` |
| 不存在返回0的字典 | `defaultdict(int)` |

## 面试话术（英文，练到张口就来）

1. Restate: "So I need to..., input is..., output is..., right?"
2. Clarify: "Can it be empty? Negatives? Duplicates?"
3. Brute force: "Let me start with a brute-force O(n²)..."
4. Optimize: "I notice... so I can bring it down to O(n) using..."
5. Code while talking out loud.
6. Test: "Let me trace through [example]..."
7. Complexity: "Time is O(n), space is O(n) / O(1)."

**卡住救生圈**：
- "Let me think out loud for a second."
- "I'm stuck on ___, but let me try ___ first."
- "Let me start with a working solution and optimize."
- "Could you give me a small hint on the direction?"
