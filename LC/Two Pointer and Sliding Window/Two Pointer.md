# 双指针 (Two Pointers) 刷题笔记

> 双指针不是一种方法，是一**大类**。底下分几种完全不同的玩法，别混。
> 核心判断：**看两个指针怎么动。**

---

## 一、两大类型（先分清用哪个）

| 类型 | 指针怎么动 | 起始位置 | 结构 | 典型题 |
|---|---|---|---|---|
| **对撞双指针** | 相向，往中间夹 | left=0, right=末尾 | 一个 while | 167, 11, 3Sum, 9 |
| **快慢双指针** | 同向，一前一后 | 都从0(或错开) | for(或while) | 26, 27, 283 |

**怎么判断用哪个：**
- 从**两端往中间夹**、每轮动一个 → **对撞**（167, 11）
- **同向**、一个探路一个收编、原地改数组 → **快慢**（26, 27, 283）

---

## 二、对撞双指针

### 模板
```python
left, right = 0, len(nums) - 1
while left < right:
    # 算点什么(和/面积/比较)
    if 条件:
        left += 1        # 移左
    else:
        right -= 1       # 移右
```

### 核心思想
- 左右从两端往中间夹，**每轮只动一个指针**
- 靠"条件"决定动谁（和小了移左、和大了移右；矮的那根移动…）
- 相遇（left >= right）就停

### 适用场景
- **有序数组**找一对数（167）
- 求**最大面积/容量**（11）
- **回文**判断（9，左右往中间比）
- 三数之和 = 固定一个 + 剩下两个对撞（3Sum）

---

## 三、快慢双指针（原地操作）

### 模板
```python
slow = 0
for fast in range(len(nums)):     # 有时 range(1, len)
    if nums[fast] 是合格的:        # ← 合格判断(看题目要什么)
        nums[slow] = nums[fast]    # 把合格的搬到slow(原地覆盖!)
        slow += 1                  # slow前进
return slow          # 或 slow+1，或 nums[:slow]
```

### 核心思想
- **slow** = "合格区的边界"（合格的东西放这儿）
- **fast** = 探路，找合格的
- fast 找到合格的 → 搬到 slow → slow 前进；不合格 → fast 跳过，slow 不动
- **`nums[slow] = nums[fast]` 是灵魂**：原地把合格元素搬到前面(覆盖不要的)，所以不用额外数组

### 那个 if 条件 = "什么算合格"（从题目翻译）
| 题 | 要什么 | 合格条件 |
|---|---|---|
| 26 去重 | 保留不重复的 | `nums[fast] != nums[fast-1]`（新值）|
| 27 移除元素 | 删掉等于val的 | `nums[fast] != val`（不是val）|
| 283 移动零 | 非零排前面 | `nums[fast] != 0`（非零）|

**怎么想到条件：看题目要什么，翻译成 if。** 要非零→`!=0`；要新值→`!=前一个`。

---

## 四、⚠️ 易错点（我踩过的坑）

1. **判断值还是下标**：判断用 `nums[fast]`，不是 `fast`（下标）！
   - ❌ `if fast != 0` → ✅ `if nums[fast] != 0`

2. **先赋值还是先++（26 vs 27 顺序不同！）**
   - **26**：`for` 从 **1** 开始，slow从0 → **先++后赋值**，return `slow+1`（slow是下标）
   - **27**：`for` 从 **0** 开始，left从0 → **先赋值后++**，return `left`（left是个数）
   - 建议统一用"先赋值后++、从0开始"版本，两题就一致，不易混

3. **原地算法不用 res**：26/27/283 直接改 nums，不用另开数组。
   - 要长度 → return 指针；要数组 → `return nums[:slow]`（切片）

4. **`nums[slow]=nums[fast]` 有作用！** 它原地把合格元素搬到前面(覆盖)。
   有时前后值碰巧一样看着没变，但整体在改数组。没它结果就错。

5. **对撞不能用在"连续子数组"**：连续子数组要保持顺序，不能排序/对撞，那是滑窗的活。

---

## 五、下标 ↔ 个数（+1 的来源）
- 下标从0开始，个数从1开始，差1
- 最后一个下标 + 1 = 个数
- 窗口长度 = `right - left + 1`（下标差+1才是元素个数）

## 七、关键感悟

1. **双指针是大类，先判断"对撞 or 快慢"**（看指针相向还是同向）。
2. **快慢指针的 if 条件 = 把"题目要什么"翻译过来**，不是背的。
3. **26/27/283 是一家**（同框架不同合格条件），一起刷才看得出联系。
4. **原地覆盖是快慢的精髓**：`nums[slow]=nums[fast]` 搬合格元素到前面，省空间。
5. **3Sum = 167的升级**：外面固定一个数，里面对撞。排序是灵魂(为了对撞+去重)。
6. **反复刷才能真会**：看懂≠会写≠真懂。要隔天默写、每遍问"为什么"。
